Game Manager + Action

Audio

Two components handle your entire audio world. GameAudioManager owns looping music and ambient tracks with automatic crossfading and global volume control. ActionPlaySound plays one-shot sound effects from any object in the scene with randomized pitch and volume variation.

Music with automatic crossfading
Ambient layer, independent of music
One-shot SFX with pitch & volume variation
Gradual volume fades via events
Getting started

Minimal setup

Music and ambient tracks are managed centrally by one GameAudioManager. Sound effects are attached to individual objects using ActionPlaySound.

  • 1
    Add GameAudioManager to a GameObject
    Create an empty GameObject named AudioManager and attach GameAudioManager. Two child AudioSources (Music Source and Ambient Source) are created automatically at runtime — you don't need to add them manually.
  • 2
    (Optional) Set a starting music clip
    Drag an AudioClip into the Starting Music Clip field and enable Play On Start to have music fade in automatically when the scene loads. Leave empty to start silent and trigger music from a UnityEvent.
  • 3
    Add ActionPlaySound for sound effects
    On any GameObject that needs to play a sound (a coin, a button, a door), add ActionPlaySound. Drag one or more AudioClips into its Audio Clips array. Wire any UnityEvent (e.g. an InputTriggerZone.onEnter) to ActionPlaySound.Play().
  • 4
    (Optional) Link ActionPlaySound to the audio manager
    Enable Route Through Audio Manager on an ActionPlaySound component to make its volume respect the SFX Volume slider on GameAudioManager. This lets a single slider control all opted-in sound effects at once — useful for a settings menu.
GameAudioManager

Music & Ambient channels

GameAudioManager maintains two independent looping audio layers. Each has its own source, volume control, and events. They never interfere with each other.

Music
Background track
Looping background music. Calling PlayMusic(clip) while music is already playing automatically crossfades to the new track — no extra setup required. Supports fade in, fade out, pause, and resume.
Ambient
Ambient layer
A second looping layer, independent of music. Use it for environmental sound (wind, rain, cave drips) that should persist regardless of what's happening with the music track. Also crossfades automatically when you call PlayAmbient(clip) while it's running.
🔀
Crossfading is automatic
If music (or ambient) is already playing when you call PlayMusic(clip), the manager creates a temporary AudioSource, fades the old track out and the new track in simultaneously, then cleans up. You never hear a hard cut. The crossfade duration is controlled by Default Fade Duration (default: 2 seconds).

Volume

All four volume sliders are independent. Master Volume controls AudioListener.volume — the global Unity volume that affects everything in the scene. The others control their respective AudioSource volumes directly. No AudioMixer is required.

SliderDefaultWhat it controls
Master Volume1Global Unity AudioListener.volume. Scales all audio in the scene.
Music Volume1Volume of the music AudioSource.
SFX Volume1Shared scalar for ActionPlaySound components with Route Through Audio Manager enabled. Has no effect on sounds that don't opt in.
Ambient Volume1Volume of the ambient AudioSource.
ActionPlaySound

Sound effects

Sound effects are handled by ActionPlaySound — a component you place directly on the object that should make the sound. Each component has its own pool of clips, randomized pitch and volume, and fires an event when it plays. Add as many as you need.

💡
One component, one AudioSource
ActionPlaySound automatically adds an AudioSource to the same GameObject if one doesn't exist. It uses PlayOneShot, so multiple calls overlap naturally — the source never interrupts itself. Multiple sounds can play from the same object at the same time.

Inspector fields

FieldDefaultWhat it does
Audio ClipsemptyArray of AudioClips to randomly choose from. One clip is picked at random each time Play() is called. Add multiple clips for natural variation (e.g. three slightly different footstep sounds).
Volume Min / Max0.8 / 1.0A random volume between these two values is chosen each time. Set both to the same value for a fixed volume. If Route Through Audio Manager is on, this value is then multiplied by the SFX Volume scalar.
Route Through Audio ManageroffWhen enabled, this component looks for a GameAudioManager in the scene and multiplies its volume by the manager's SFX Volume. Lets a single slider control all opted-in effects at once.
Output Mixer GroupnoneOptional AudioMixerGroup to route this sound through (e.g. SFX, Music, Ambience). Leave empty to use the default output.
Pitch Min / Max1.0 / 1.0A random pitch between these two values is chosen each time. Values below 1 lower the pitch; values above 1 raise it. Try 0.9–1.1 for subtle variation, 0.8–1.2 for noticeable randomness.

Events

EventParametersWhen it fires
onPlay—Fires each time a clip is successfully played. Wire to a visual flash, particle burst, or animation trigger.

Methods

MethodParameterWhat it does
Play()—Picks a random clip from the array, applies random pitch and volume, and plays it. Wire this to any UnityEvent source such as InputKeyPress or InputTriggerZone.
SetVolume()float 0–1Sets a fixed volume with no variation (both min and max are set to the same value). Useful for a volume slider in a settings UI.
SetPitch()float 0.1–3Sets a fixed pitch with no variation (both min and max are set to the same value).
Wiring behavior

Events

Wire these to UI, lighting, or other managers to react to audio state changes without polling.

GameAudioManager events

EventParametersWhen it fires
onMusicStarted—Music begins playing (or crossfade completes). Wire to a "Now Playing" UI update or a synced visual effect.
onMusicStopped—Music finishes fading out and the source stops. Wire to trigger silence-based effects.
onAmbientStarted—Ambient track begins (or ambient crossfade completes).
onAmbientStopped—Ambient track stops after fading out.
onVolumeChangeComplete—Fires when any gradual volume change (SetMusicVolumeGradual, SetMasterVolumeGradual, etc.) finishes. Wire to trigger the next audio action in a sequence.

ActionPlaySound events

EventParametersWhen it fires
onPlay—Fires each time Play() successfully plays a clip. Wire to particles, animation, or other feedback.

Music methods

MethodParameterWhat it does
PlayMusic()AudioClipPlays a music track with fade in. If music is already playing, crossfades automatically using the default fade duration.
StopMusic()—Fades music out over the default duration, then stops. Fires onMusicStopped when complete.
PauseMusic()—Pauses music instantly, preserving playback position.
ResumeMusic()—Resumes music from where it was paused.

Ambient methods

MethodParameterWhat it does
PlayAmbient()AudioClipPlays an ambient track with fade in. If ambient is already playing, crossfades automatically.
StopAmbient()—Fades ambient out and stops. Fires onAmbientStopped when complete.
PauseAmbient()—Pauses ambient playback, preserving position.
ResumeAmbient()—Resumes ambient from where it was paused.

Volume methods

MethodParameterWhat it does
SetMasterVolume()float 0–1Sets AudioListener.volume instantly, scaling all audio in the scene.
SetMusicVolume()float 0–1Sets music AudioSource volume instantly.
SetSFXVolume()float 0–1Sets the shared SFX scalar instantly. Only affects ActionPlaySound components with Route Through Audio Manager on.
SetAmbientVolume()float 0–1Sets ambient AudioSource volume instantly.
SetMasterVolumeGradual()float 0–1Fades AudioListener.volume to the target over the default fade duration. Fires onVolumeChangeComplete when done.
SetMusicVolumeGradual()float 0–1Fades music AudioSource volume to the target over the default fade duration. Fires onVolumeChangeComplete when done.
SetSFXVolumeGradual()float 0–1Fades the shared SFX scalar to the target over the default fade duration. Fires onVolumeChangeComplete when done.
SetAmbientVolumeGradual()float 0–1Fades ambient AudioSource volume to the target over the default fade duration. Fires onVolumeChangeComplete when done.
StopAllAudio()—Stops music and ambient immediately with no fade. Useful for game over or scene transitions.
